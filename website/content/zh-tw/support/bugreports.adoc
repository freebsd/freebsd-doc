= 問題回報

== 問題回報資料庫

* https://bugs.FreeBSD.org/search/[搜尋目前的問題回報]
* https://bugs.FreeBSD.org/submit/[送出新的問題回報]

問題回報，或稱為 _PR_，可以是任何與 FreeBSD 相關的問題，包括作業系統程式的問題、文件錯誤或是提交者希望有的新功能。

問題回報始於狀態 _new（新）_ ，問題解決後則會被關閉。每個PR都會被賦予一個數字 ID 以利追蹤。許多 FreeBSD 的修正都會提及相對應的 PR。

== 如何送出問題回報

問題回報是以 https://bugs.FreeBSD.org/submit/[網路表單] 傳送給開發團隊。送出 bug 前要先建立一個帳號。請留意的是寄給郵件論壇的訊息並_不會_以官方的問題回報系統追蹤，而且有可能被淹沒而消失！

提交問題回報前，請先閱讀 link:{problem-reports}[撰寫 FreeBSD 問題回報] 這篇文章。此文敘述何時該提交問題回報，該包含甚麼內容。link:{pr-guidelines}[問題回報處理守則]亦提及一些有用的相關資訊。

FreeBSD 問題回報系統使用 http://www.bugzilla.org/[Bugzilla] 來追蹤。
